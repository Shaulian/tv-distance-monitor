Methodology — Philosophy

  • Tests verify behavior, not implementation. A failing test should tell you what broke, not how.
  • Every bug means a missing test. Add it before the fix; land it red, then green, on the same branch.
  • Mock only at system boundaries — camera hardware, audio hardware, file I/O. Never mock the logic under test.
  • Fail loud, not silent. For a child-safety product, ambiguity must trigger an alert, never suppress one. ADR-016
  • Coverage is necessary but not sufficient. Mutation testing + property tests verify the suite would catch a behavior change, not just that it ran the line.

  • Unit + integration + perf all green
  • Core ≥ 90% / floor ≥ 55% branch coverage

Coverage — How We Calculate It

  • Branch coverage, not line coverage. Uncovered else / elif paths count as misses. ADR-014
  • Scope: source = ["."] with omit = ["tests/*", "venv/*", "tvdm.spec"]. No omit of product code.
  • Measured on pytest tests/unit/ tests/integration/ --cov — both tiers contribute.
  • Two gates, both blocking on every PR:
GateScopeThresholdToday
Core-logic gate audio/ camera/ config/ detection/ state.py ≥ 90% 96%
Full-codebase floor everything (incl. main.py + tray/*) ≥ 55% 57%
The floor ratchets up; never down. Lowering it requires a new ADR.

Deep Dive — The Stereo Depth Pipeline

Why this area: it's the product's one job. Compute how close a person is to the TV. If this is wrong, the alert never fires — a silent safety failure.

What we found: during the QA review, a P1 functional defect caused the pipeline to report a fixed distance regardless of the person's actual position. The unit suite was 100% green. The defect lived in the orchestration glue, which was at 0% coverage.

Story QA.WS1 — ADR-015 — PR #2 + #3.

The Bug — Shared PersonDetector


# main.py (pre-fix) — one detector for both cameras
detector = PersonDetector()
threading.Thread(target=_camera_loop, args=(
    camera_manager, FrameProcessor(),
    detector,                # ← passed once
    depth_estimator, ...)).start()

# _camera_loop body
dets_l = detector.detect(left_n)     # call A
dets_r = detector.detect(right_n)    # call B
distance = depth_estimator.estimate_distance(dets_l, dets_r)

PersonDetector processes every 3rd frame and returns a cached result on the rest. Two calls per iteration on the same instance land on consecutive _frame_count values — so the second call returns a cached copy of the first.

What That Means in Numbers

Across 4 iterations of the live loop:

IterLeft detectionRight detectionDisparityComputed dist.
1L₁ (HOG)cached L₁0intercept
2cached L₁R₁ (HOG)real ✓plausible
3cached R₁cached R₁0intercept
4L₂ (HOG)cached L₂0intercept

3 of every 4 iterations report distance = the calibration intercept (a constant). The product cannot tell where the person actually is. For a child-safety device, this is a P1 — the failure direction is silent.

Why the Unit Suite Missed It

What the unit tests checked

  • PersonDetector: fresh instance per test, HOG mocked — cache never crosses cameras
  • DepthEstimator: hand-built detection tuples — never sees the corrupted dets
  • The _camera_loop wiring lived in main.py at 0% coverage

What the strategy doc had named

  • tests/integration/test_camera_to_depth_pipeline.py
  • Folder existed; file did not.
  • The integration tier was an aspiration, not a gate.
The defect fell straight through the missing tier. The unit pyramid base was strong; the integration mid-section was empty; the orchestration glue at the top was uncovered.

The Fix — One Detector Per Camera


# main.py (post-fix) — per-camera detectors
threading.Thread(target=_camera_loop, args=(
    camera_manager, FrameProcessor(),
    PersonDetector(),       # detector_left
    PersonDetector(),       # detector_right  ← ADR-015
    depth_estimator, ...)).start()

Same fix applied to StereoCalibrator.calibrate_diamond — the calibration curve was also being fit from corrupted disparities.

The integration test that proved both

  • Behavioural: distinct left/right detections → correct distance; with the bug, person_too_close=True (false alarm); after fix, False.
  • Structural guard: inspect.signature(_camera_loop) must contain detector_left and detector_right. A revert fails at collection time.

Property Tests — Pressure on the Fail-Safe Contract

Hypothesis generates random inputs and asserts invariants, not specific outputs.

Key invariant (ADR-016): if at least one camera sees a person, the verdict is never silent.


@given(
    left=st.lists(_detection_pair_strategy, min_size=1, max_size=5),
    right=st.lists(_detection_pair_strategy, min_size=1, max_size=5),
    min_safe=_min_safe_strategy,
)
def test_assess_proximity_with_any_detections_is_never_silently_safe(left, right, min_safe):
    estimator = DepthEstimator({"slope": 0.04, "intercept": 0.5})
    _too_close, reason = estimator.assess_proximity(_dets(left), _dets(right), min_safe)
    assert reason != "no_person"   # ADR-016 — fail loud, not silent

Hypothesis exercises this with hundreds of generated inputs per run. If a future change ever introduces a silent-failure path for some unusual input the example tests didn't think of, this test finds it.
